# 对话功能设计文档

## 需求

Claude Code 可以向用户提问，用户在手机上输入文字回复，Claude 收到回复后继续执行。

---

## 使用场景示例

### 场景：Claude 需要用户做决策

```
Claude: 需要确定按钮颜色，让我问一下用户...

执行: bash ask.sh "按钮应该用什么颜色？" "用户在之前提到想要一个更醒目的设计"

手机端显示:
┌─────────────────────────────────┐
│  💬 Claude 提问                 │
├─────────────────────────────────┤
│  按钮应该用什么颜色？            │
│                                 │
│  说明: 用户在之前提到想要一个    │
│       更醒目的设计               │
│                                 │
│  ┌───────────────────────────┐ │
│  │                           │ │
│  │ 输入你的回复...            │ │
│  │                           │ │
│  └───────────────────────────┘ │
│                                 │
│           [发送]                │
└─────────────────────────────────┘

用户输入: 用蓝色，和主题色保持一致

Claude 收到: "用蓝色，和主题色保持一致"
Claude: 好的，使用蓝色按钮...
```

---

## 两种请求类型对比

| 类型 | 用途 | 手机端显示 | Claude 收到的结果 |
|------|------|-----------|------------------|
| **审批** (approval) | 执行危险命令前确认 | ✓ 批准 / ✗ 拒绝 按钮 | approved 或 rejected |
| **提问** (question) | 需要用户输入信息 | 文字输入框 + 发送按钮 | 用户输入的文字 |

---

## 技术方案

### 1. 数据模型

请求对象新增字段：

```javascript
{
  id: "abc123",
  type: "approval" | "question",  // 新增：请求类型
  command: "...",                  // 审批=命令，提问=问题内容
  description: "...",              // 补充说明
  risk: "normal",                  // 仅审批用，提问忽略
  status: "pending" | "approved" | "rejected" | "replied",  // 新增 replied
  reply: "用户的回复文字",          // 新增：用户回复内容
  created_at: "...",
  decided_at: "..."
}
```

### 2. API 变更

#### 创建请求 - POST /api/request

新增 `type` 字段：

```json
// 审批请求（现有，不变）
{ "command": "rm -rf /tmp", "description": "清理", "risk": "danger" }

// 提问请求（新增）
{ "command": "按钮用什么颜色？", "description": "设计决策", "type": "question" }
```

#### 回复问题 - POST /api/reply（新增）

```json
// 请求
{ "id": "abc123", "message": "用蓝色" }

// 响应
{ "ok": true, "request": { "id": "abc123", "status": "replied", "reply": "用蓝色", ... } }
```

### 3. 手机端 UI

#### 审批类型（现有）
- 显示命令、描述、风险等级
- 底部两个按钮：✗ 拒绝 / ✓ 批准

#### 提问类型（新增）
- 显示问题内容、说明
- 底部：文字输入框 + 发送按钮
- 输入框可多行，有 placeholder "输入你的回复..."

### 4. 新脚本：ask.sh

```bash
# 用法
bash ask.sh "按钮用什么颜色？" "设计决策说明" [超时秒数]

# 示例
bash ask.sh "按钮用什么颜色？" "用户想要更醒目的设计" 300

# 返回值
# - stdout 输出用户的回复文字
# - exit 0 = 收到回复
# - exit 2 = 超时
```

### 5. 历史记录

- 审批：显示「✓ 已批准」或「✗ 已拒绝」
- 提问：显示「💬 已回复」+ 回复内容
